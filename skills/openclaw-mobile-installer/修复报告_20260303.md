# OpenClaw 手机安装脚本修复报告

**修复时间**: 2026-03-03 15:45
**问题报告**: 用户反馈安装时卡在"NODE.JS 版本太低"

---

## 🔍 问题分析

### 原因

1. **Termux 自带 Node.js 版本过旧**
   - Termux 的 `nodejs-lts` 包可能只有 v14-v16
   - OpenClaw 要求 Node.js 22+

2. **Ubuntu proot 默认源 Node.js 版本低**
   - Ubuntu 默认 apt 源的 nodejs 包版本较老
   - 未指定版本时会安装旧版

---

## ✅ 修复内容

### 1. Step 2: Termux 环境检查

**修改前**:
```bash
pkg install -y nodejs-lts python git curl wget proot-distro
```

**修改后**:
```bash
pkg install -y nodejs-lts python git curl wget proot-distro
NODE_VERSION=$(node --version | cut -d'v' -f2 | cut -d'.' -f1)
if [ "$NODE_VERSION" -lt 22 ]; then
    print_warning "Termux Node.js version is $NODE_VERSION (< 22), OpenClaw will run in Ubuntu proot with Node.js 22+"
else
    print_success "Termux Node.js version: $(node --version)"
fi
```

**效果**: 明确告知用户 Termux 的 Node.js 版本，避免混淆

---

### 2. Step 4: Ubuntu proot 安装 Node.js 22+

**修改前**:
```bash
apt install -y nodejs npm
npm install -g openclaw
```

**修改后**:
```bash
# Install Node.js 22+ (required for OpenClaw)
apt install -y curl
curl -fsSL https://deb.nodesource.com/setup_22.x | bash -
apt install -y nodejs

# Verify Node.js version
NODE_VERSION=$(node --version | cut -d'v' -f2 | cut -d'.' -f1)
if [ "$NODE_VERSION" -lt 22 ]; then
    echo "[ERROR] Node.js version must be 22 or higher, current: $NODE_VERSION"
    exit 1
fi
echo "[OK] Node.js $(node --version) installed"

# Install OpenClaw
npm install -g openclaw
```

**效果**: 
- ✅ 强制安装 Node.js 22.x LTS
- ✅ 版本验证，低于 22 直接报错退出
- ✅ 明确的成功/失败提示

## 🐛 二次修复 (2026-03-03 15:58)

**问题**: 用户反馈第 175 行错误

**原因**: heredoc 语法错误，`STARTUP_EOF` 和 `CRON_EOF` 没有加引号，导致变量被提前展开

**修复**:
```bash
# 修改前（错误）
cat > "$STARTUP_FILE" << STARTUP_EOF

# 修改后（正确）
cat > "$STARTUP_FILE" << 'STARTUP_EOF'
```

**影响**: 
- 不加引号：`$WORKSPACE_DIR` 和 `$HIJACK_FILE` 会在脚本生成时展开（为空值）
- 加引号：保留变量，在 boot.sh 运行时展开
